1. Introduction

SCEP Bus App provides real-time bus tracking, attendance management, and safety alerts for students, parents, and faculty of Satpuda College of Engineering and Polytechnic.

2. Information We Collect

  • Student GPS location (real-time and background tracking while bus tracking is active)
  • Bus GPS location from tracking devices
  • Name, email, mobile number
  • Attendance data (boarding and arrival)
  • Complaint and support ticket data
  • Device token (FCM for push notifications)
  • Bus pass photo (uploaded for verification purposes)
  • Payment transaction data (processed via Razorpay)
  • Device identifiers (for advertising via Google AdMob)

3. How We Use Data

  • Live bus tracking and ETA calculation
  • Attendance marking (boarding & arrival detection via background GPS)
  • Bus start alerts and 5 KM proximity notifications
  • Admin dashboard analytics
  • Complaint and support ticket management
  • Bus pass photo verification
  • Payment processing for photo change fees
  • Personalized in-app advertising (Google AdMob)

4. Background Location

This app collects GPS location in the background only while the bus tracking feature is active. Background location is used exclusively to:

  • Provide real-time bus ETA and proximity alerts
  • Automatically detect boarding and arrival for attendance
  • Enhance student safety by alerting when the bus is nearby

Background location collection stops automatically when the bus is idle or the tracking service is not active. Your location is never sold to any third party.

5. Data Sources (GPS Providers)

We receive bus location data from third-party GPS tracking providers:

  • Volty GPS API
  • SML Saarthi GPS System

These services are used only for tracking bus movement and improving transport safety.

6. Third-Party Services & Data Sharing

We do not sell user data. The following trusted third-party services process data on our behalf:

  • Google Firebase — Authentication, database, cloud messaging, storage
  • Google Maps SDK — Map display and route visualization (sends device location to Google)
  • Google AdMob — In-app advertising (may collect device identifiers and ad interaction data)
  • Razorpay — Payment processing for photo change fees (collects payment card/UPI data)
  • Redis — Temporary server-side caching (not shared externally)

Each third-party service is governed by its own privacy policy.

7. Data Storage & Security

  • Data stored securely in Firebase database
  • Temporary cache stored in Redis
  • Access restricted to authorized college administrators

7. User Control

Users can request deletion of their data through college administration. Direct self-deletion is not available inside the app.

8. Permissions

  • Location access (for bus tracking)
  • Notification access (for alerts)
  • Internet access (for real-time updates)

9. Complaints System

Users can submit complaints which are securely stored and reviewed by the college authority.

10. Children’s Privacy

This app can be used by students, parents, and faculty. No harmful or sensitive content is collected.

11. Data Usage Purpose

  • Bus tracking & safety alerts
  • Attendance automation
  • Transport optimization
  • Analytics for college administration
